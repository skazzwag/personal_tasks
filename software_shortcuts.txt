How to set up SSH key for signing commits

Step 1 - Generate SSH key for commit signing (ed25519 is a public-key cryptographic algorithm)
$ ssh-keygen -t ed25519 -C "your@email.com" -f ~/.ssh/id_ed25519_repo-signing

Step 2 - add as a signing key on GitHub
Go to https://github.com/settings/ssh/new and change key type to "Signing Key"
Run the following command in terminal: cat ~/.ssh/id_ed25519_repo-signing.pub
Copy and paste into GitHub new SSH key page

Step 3 - configure git to use this SSH key for signing (think you can add the --global flag to have it done everywhere)
$ git config gpg.format ssh
$ git config user.signingkey "$(cat ~/.ssh/id_ed25519_repo-signing.pub)"
$ git config commit.gpgsign true

Can confirm step 3 is done with the following
$ git config --get commit.gpgsign
$ git config --get user.signingkey

Step 4 - Need to make sure that SSH Agent is running and the key is added (commit signing won't work without it)
$ eval "$(ssh-agent -s)"

Can confirm this with 
$ ssh-add -l

Add the key with the following
$ ssh-add ~/.ssh/id_ed25519_repo-signing

Optional Step - Make SSH agent start automatically in terminal by adding the following to .bashrc or .bash_profile

# Start ssh-agent if not running
if ! pgrep -u "$USER" ssh-agent > /dev/null; then
  eval "$(ssh-agent -s)"
fi

# Add signing key
ssh-add -q ~/.ssh/id_ed25519_repo-signing 2>/dev/null
